ENCM 369

Computer Organization

A picture of the PIC microcontroller that was used in the course labs.

If I had to sum up this course in one sentence, it would be that you learn how the individual components of a computer are put together to actually make it work. Up until this point, I had learned about the parts that made up a computer such as logic gates and multiplexors, but I had no idea how they all fit together to create a working CPU. I was surprised to learn that it was more complex than I thought, and that there was a lot of engineering to be done to make computers as fast as they are in our day and age.

One big thing I learned that I found really applicable was how to program a microcontroller. We learned from scratch how to both build and enter the firmware into our circuits to make them perform specific tasks. This was applicable to my ENEL 300 project because we needed a microcontroller to be the brain of our product. Luckily, my previous experience with C helped me immensely because the PIC microcontrollers we used were very low-level. Learning how to program a low-level microcontroller such as the PIC helped me learn how to code more modern microcontrollers like Arduino much faster!
